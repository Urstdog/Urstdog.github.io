2.原因分析

在MySQL中，当字符串和数字进行比较时，会发生隐式转换。MySQL会尝试将字符串转换为数字，然后再进行比较。这个转换遵循一些规则：

  1. 如果字符串以数字开头，那么将会被转换为这个数字，直到遇到非数字字符为止。例如，字符串 '123abc' 会被转换为数字 123。
  2. 如果字符串不能转换为有效的数字，那么将会被转换为0。
  3. 如果字符串以空格开头，会被忽略。
  4. 如果字符串以其他非数字字符开头，无法转换为数字，将会被转换为0。

示例：

  • '123' = 123 会被视为相等。
  • 'abc' = 0 字符串 'abc' 会被转换为0，所以相等。
  • '456xyz' = 456 字符串 '456xyz' 会被转换为 456，所以相等。

这种隐式转换的行为可能会导致一些意想不到的结果，因此在比较时最好确保比较的两个值的类型

由于setmeal_id字段转换成数字比较大,会出现精度损失

换成字符串类型就可以查询到正常结果
