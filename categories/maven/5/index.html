maven依赖管理

发表于2024-09-28|更新于2024-09-28|maven
|字数总计:251|阅读时长:1分钟|阅读量:

1. 依赖配置

  1. 依赖 : 指当前项目运行所需要的jar包,一个项目可以引入多个依赖

  2. 配置 :

  1. 在 pom.xml 中编写 标签
  2. 在 标签中 使用引入坐标
  3. 定义坐标的 groupId , artifactId , version
  4. 点击刷新按钮,引入新坐标

2. 依赖传递

依赖具有传递性

3. 排除依赖

主动断开依赖资源,别排除的资源无需指定版本

4. 依赖范围

依赖jar包,默认情况下,可以在任何地方使用.可以通过设置作用范围.

作用范围 :

  1. 主程序有效 (main文件夹)
  2. 测试程序有效 (test文件夹)
  3. 是否参与打包 (package指令范围)

5. 生命周期

生命周期就是为了对所有的Maven项目构建过程进行抽象和统一

maven 中有3套相互独立的生命周期 :

  1. clean 清理工作
  2. default 核心工作 , 如 : 编译,测试,打包,安装,部署 等
  3. site 生成报告,发布站点 等
